# Changelog

##  [3.1.0] — 2026-06-28

### Исправлено: Command/Command.php

- **Удалён неиспользуемый импорт** `Pet\Apache\Apache` — класс не существует в проекте.
- **Удалена конфликтующая константа класса** `const ROOT = ROOT . DIRECTORY_SEPARATOR` — переопределяла глобальную константу `ROOT`, что вызывало фатальную ошибку. Константа нигде не использовалась.
- **Исправлена логическая ошибка с индексами массива** — `unset($comm[0])` переиндексировал массив, после чего обращения к `$comm[1]` и `$comm[2]` указывали не на те элементы. Теперь имя команды и аргумент извлекаются до удаления элемента.
- **Удалена неиспользуемая переменная** `$inCommands`.
- **Добавлена полная типизация** свойств и методов (`string`, `array`, `self`, `void`, `?string`).
- **Добавлена валидация в `startSocket()`** — проверка на `null` аргумент и на существование файла сокета перед `include`.

### Переработано: Errors/AppException.php

- **Удалено избыточное свойство `$errorCode`** — родительский класс `Exception` уже хранит код ошибки в `$this->code`, а метод `getCode()` доступен по наследству. Метод `getErrorCode()` удалён как дублирующий.
- **Добавлена полная типизация** — `string $message`, `int $code`, `?Throwable $previous`.
- **Заменён `Exception $previous` на `?Throwable $previous`** — соответствует современному PHP 8+.
- **Добавлен PHPDoc** с описанием класса и параметров конструктора.

### Переработано: Errors/Errors.php

- **Добавлена фильтрация по типам ошибок** — теперь логируются только фатальные ошибки (`E_ERROR`, `E_PARSE`, `E_CORE_ERROR`, `E_COMPILE_ERROR`, `E_USER_ERROR`), а не все подряд.
- **Добавлен метод `errorTypeName()`** — человекочитаемые имена типов ошибок через `match()`.
- **Добавлена полная типизация** — `void`, `string`, `int`, `array` для всех методов и свойств.
- **Константа `LOG` сделана `private`** — инкапсуляция внутренней реализации.
- **Добавлен PHPDoc** для класса и всех методов.
- **Упрощён конструктор** — вызов `register_shutdown_function` без лишнего метода-обёртки.
